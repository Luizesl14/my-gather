# 49 — Observability & Definition of Done Spec

## Objetivo

Definir logs, métricas mínimas e pronto para entrega.

## Logs backend

- Boot.
- Request id.
- Erro HTTP.
- WebSocket connect/disconnect.
- Evento realtime rejeitado.
- Movimento inválido.

## Métricas futuras

- Usuários online.
- Eventos por segundo.
- Latência WebSocket.
- Erros por rota.

## Definition of Done

- Código compila.
- Testes relevantes passam.
- API/evento documentado.
- DDD respeitado.
- Light/dark verificado.
- Animações verificadas.
- Sem segredo versionado.
- Erros tratados.

## Critérios de aceite

- Cada entrega informa specs atendidas.

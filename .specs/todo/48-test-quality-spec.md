# 48 — Test & Quality Spec

## Objetivo

Definir testes mínimos por fase.

## Backend

- Domínio: unitários.
- Use cases: unitários.
- REST: integração.
- WebSocket: integração.
- Banco: migration e repository tests.

## Web

- Theme: snapshots/estado.
- Widgets: toolbar, chat, card, modal.
- Game UI: renderer e movimento.
- Realtime: mocks de eventos.

## E2E MVP

- Dois usuários no escritório.
- Movimento realtime.
- Mesa/chamada/recado.
- Sala/chat/typing.
- Gestos/reacões.

## Critérios de aceite

- Testes críticos passam antes de fechar MVP.

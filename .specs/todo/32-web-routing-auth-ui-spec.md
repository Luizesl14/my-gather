# 32 — Web Routing & Auth UI Spec

## Objetivo

Criar rotas e telas iniciais de autenticação.

## Rotas

- `/login`
- `/organizations`
- `/workspaces`
- `/office/:workspaceId`
- `/avatar`
- `/admin`

## Telas

- Login.
- Seleção de organização.
- Seleção de escritório.

## Critérios de aceite

- Rotas nomeadas funcionam.
- Login consome API quando backend existir.
- Estados loading/error/empty existem.
